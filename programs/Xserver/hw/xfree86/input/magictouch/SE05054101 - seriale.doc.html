1.Data Formats


RS-232 mode:
Connect to serial port (RS-232):
When reset, Touch pad data format is compatible to Msmouse.

When host send '00h' with 9600 bps, 8bit data, 1 stop, no parity;
Pad will be set to 9600bps, and return '0Fh'.

Data format in 9600bps:
Data format:
Byte 1:D7D6D5D4D3D2D1D0　　
　0 　 10C 　 M　L 　 R　P

　　Bit0　“1”= Touch, “0”=Un-touch　(P)
1 R-key: “1”= Lift key depressed.(R)
2 L-key: “1”= Right key depressed.(L)
3 M-key: “1”= Middle key depressed.(M)
4 Click status: “1”= user click the pad.(C)
5 Always “0”
6 Always “1”
7 Always “0”

D7D6D5D4D3D2D1D0
Byte 2: 0　0　0　0　x9　 x8　x7 　x6 High 4 Bit of X data
Byte 3: 0　0x5x4x3 　x2　x1 　x0Low 6 Bit of X data
Byte 4: 0　0　0　0　y9 　y8　y7 　y6High 4 Bit of Y data
Byte 5: 0　0y5y4y3y2　y1 　y0Low 6 Bit of Y data

  TOUCH RANGE:                                
       X=60 - 960  Y=60 - 960                      
  SAMPLE RATE:                                
      120 POINT/SEC                              

Pen  mode:
     data format is same, but move is more fast, and no click check.




2.Function Command
Function：

Provides user use the driver that develop by yourself to control Super Elite’s
Touch function Chip Controller.
No more need to use Super Elite driver to controller.

[Attention]:
If all commands exceed 2 bytes, you must input 1 byte by 1 byte,
In other words, you must input 1 byte and receive ECHO of Chip
Controller then input next byte.
Otherwise, maybe confuse whole system.























2.1Function Description


1. [00H]
Demand the Chip Controller doing "Software Reset"process or identify
Chip Controller insists or not.


Return:0FH
Software Reset OK

Example:
Driver Command:00H
Chip Controller Return:0FH



2. [01H]
Read EEPROM Data

Entry:A1A201H
A1: 30H ~ 33H
A2: 30H ~ 3FH

Return:Coordinate (A1, A2) Data

Example:
Driver Command:30H30H01H
(Read coordinate (0,0) address date of EEPROM)
Chip Controller Return:08H21H
(Represent coordinate (0,0) address in EEPROM)

Note:
Each address of EEPROM can store 1word data; totality can store 64 words,
Coordinate Data show from (00H, 00H) to (03H, 0FH),
Indicate from (30H, 30H) to (33H, 3FH) in Chip controller at present.



3. [02H]
Write data into EEPROM

Entry:A1A2D1D2D3D402H
A1: 01H
A2: 01H
D1 ~ D4: 30H ~ 39H
Note:


Above data description is entry method of IC AT93C46,if you use IC


NMC9346N,you must entry 0CHA1A20BHA1A2D1D2D3D402H

Return:ECHO transport command
Example:
Driver Command:30H30H31H38H30H31H02H
(Write 1801 data into (0,0) address)
Chip Controller Return:30H30H31H38H30H31H02H
(ECHO transport command 1 byte by 1 byte)


4. [03H]
Change the return data of Panel from Chip Controller to A/D Data

Return:03H normally

Example:
Driver Command:03H
Chip Controller:03H


5. [04H]
Change the return data of Panel from Chip Controller to the set-up value

Return:04H

Example:
Driver Command:04H
Chip Controller:04H



6. [06H]
Read the Jump set status of Chip Controller

Return:setting Data
Set up as below:
D7, D6:Don't Care
D5:Jump 2
D4:Jump 1
D3:STREAM / POINT MODE
D2D1D0:Baud Rate
1:1200
2:2400
3:4800
4:9600
Example:
Driver Command:06H
Chip Controller:ECH


7. [07H]
Transform the Value of Chip Controller to Point Mode

Return:07H
Example:
Driver Command:07H
Chip Controller:07H


8. [08H]
Transform the Value of Chip Controller to STREAM Mode

Return:08H
Example:
Driver Command:08H
Chip Controller:08H





9. [09H]
Command the Chip Controller to stop reading Panel Data temporarily

Return:09H
Example:
Driver Command:09H
Chip Controller:09H


10. [0AH]
Chip Controller regains to read Panel data

Return:0AH normally
Example:
Driver Command:0AH
Chip Controller:0AH


11. [0BH]
Clear the Position Data of EEPROM

Entry:A1A20BH
Return:0BH
Example:
Driver Command:30H30H0BH
(Clear coordinate (0,0) address data of EEPROM)
Chip Controller:30H30H0BH
(ECHO transport 1 byte by 1 byte)


12. [0CH]
Clear the RAM Buffer of Chip Controller That communicate with EEPROM

Return:0CH
Example:
Driver Command:0CH
Chip Controller:0CH


13. [11H]
Transform Baud Rate to 1200 bps

Return:11H of Baud Rate 1200 normally
Example:
Driver Command:11H
Chip Controller:11H (Baud Rate: 1200 bps)


14. [12H]
Transform Baud Rate to 2400 bps

Return:12H of Baud Rate 2400 normally
Example:
Driver Command:12H
Chip Controller:12H (Baud Rate: 2400 bps)


15. [13H]
Transform Baud Rate to 4800 bps

Return:13H of Baud Rate 4800 normally
Example:
Driver Command:13H
Chip Controller:13H (Baud Rate: 4800 bps)


16. [14H]
Transform Baud Rate to 9600 bps

Return:14H of Baud Rate 9600 normally
Example:
Driver Command:14H
Chip Controller:14H (Baud Rate: 9600 bps)






17. [1FH]
Enter sleep mode

Return:1FH normally
E0H while awake
(Whether use Touch function or Keyboard input)
Example:
Driver Command:1FH
Chip Controller:1FH



18. [30H ~ 3FH]
Set the coordinate address of EEPROM,2 Bytes regard as 1 set,
The same description of A1, A2 with2. [01H]

Entry:A1A201H
A1:30H ~ 33H
A2:30H ~ 3FH

Return:ECHO transport Command
Example:
Driver Command:30H30H
(Set coordinate (0,0) address data)
Chip Controller:30H30H
